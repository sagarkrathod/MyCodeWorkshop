2. UI need to read this object and find if there are any messages exist to display.

3. If message found, read its property called 'messageType' and define CSS style accordingly

    like for type "INFO" show above kind of style for MCOM.

4. The styles & icons are hard coded in UI. Only the message will come from ShopApp as following string. where Tag P is heading, Tag span is body & tag A is link

   <p>We've updated your Wallet with your new Macy’s Credit Card number.</p><span>Your offers & star passes will work with your new 16-digit chip card. Just start using your new card and destroy the old one.</span></br><a href='http://www.macys.com/'>find out more</a>

5. styles for P, span and a tag will come from UI (see code for above messages)

6. There are not more than one message to show at this point, But MCOM is making code in such a way that it could handle multiple message to display at the same time if there is such requirment comes in future.
